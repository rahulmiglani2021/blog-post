10 GitHub Tricks You Should Know

This tutorial explains how to import a new project into Git, make changes to it, and share changes with other developers.   If you’re aiming to become a certified coder, you should know the following git tricks. These tricks have been helpful in improving my productivity, and hopefully they’ll help you too.

1. Git clone

git clone <name-of-the-repository-link>

2. Git branch

git branch <branch-name>

3. Push the new branch into the remote repository

git push -u <remote> <branch-name>

4. Viewing branches:

git branch or git branch --list

5. Deleting a branch:

git branch -d <branch-name>

6. To checkout and switch between branches

git checkout <name-of-your-branch>

7. To check status

git status

8. To add a single file and to add everything and once

git add <file>   git add -A

9. most-used command of Git. When you need to Save changes:

git commit -m "commit message"

10. After committing your changes, the next thing you want to do is send your changes to the remote server:

git push <remote> <branch-name>
